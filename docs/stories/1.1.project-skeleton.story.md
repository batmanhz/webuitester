# Story 1.1: Project Skeleton & Core Dependencies
**Status**: Draft

## Story
**As a** Developer,
**I want** to initialize the project repository with a monorepo structure, backend (FastAPI), frontend (Vue 3), and basic Agent module,
**so that** we have a stable foundation for building features.

## Acceptance Criteria
1.  Root directory structure created as per Architecture.
2.  Backend: FastAPI installed, running, and accessible at `http://localhost:19000/health`.
3.  Frontend: Vue 3 + Vite project created, running at `http://localhost:5173`.
4.  Agent: Basic Python module structure created.
5.  Pre-commit hooks or basic linting configured (Ruff/Eslint).

    - [ ] Create `.gitignore` (Python/Node/IDE defaults)

    - [ ] Create `package.json` for frontend (if not handled by `npm create vite`)

## Dev Notes
### Architecture Alignment
*   **Project Structure**:
    *   `backend/`: FastAPI application
    *   `frontend/`: Vue 3 application
    *   `agent/`: Core Agent logic (Python)
    *   `docs/`: Documentation (Already exists)
*   **Technology Stack** [Source: architecture.md#7]:
    *   Backend: Python 3.10+, FastAPI
    *   Frontend: Vue 3, Vite, Element Plus, Pinia
*   **API**:
    *   Backend will serve API at `/api` (or default `/` for now)
    *   Frontend will consume API (Proxy setup in Vite config recommended later)

### Testing Standards
*   **Backend**: `pytest` will be the standard (install it now).
*   **Frontend**: `vitest` (install it now).
