# Story 2.1: Test Case Editor UI

## Description
Develop the frontend interface for creating and editing test cases. This connects the Frontend to the Backend CRUD API developed in Story 1.2.

## Acceptance Criteria
1.  **List View**:
    *   Display a list of test cases (ID, Name, URL).
    *   "New Case" button to open the creation form.
2.  **Editor Form**:
    *   Input fields for **Test Case Name** and **Target URL**.
    *   **Steps List**: Ability to add, remove, and reorder steps.
    *   Each step has: **Instruction** (Text) and **Expected Result** (Text).
3.  **API Integration**:
    *   Saving the form sends a `POST` request to `/api/cases`.
    *   Opening an existing case sends a `GET` request to `/api/cases/{id}`.
4.  **Validation**: Basic form validation (required fields).

## Technical Notes
*   Use Vue 3 Composition API.
*   Use Element Plus `el-form`, `el-table`, and `el-input`.
*   Store state in a Pinia store or local component state.
