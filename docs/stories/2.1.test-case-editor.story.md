# Story 2.1: Test Case Editor UI (2-Step Workflow)

## Description
Develop the frontend interface for the new 2-step test case creation workflow. This involves an "Intent Input" phase followed by an "AI Generation" phase, and finally a "Manual Refinement" editor.

## Acceptance Criteria
1.  **List View**:
    *   Display a list of test cases (ID, Name, URL).
    *   "New Case" button to open the **Intent Input Dialog/Page**.
2.  **Phase 1: Intent Input**:
    *   Input field for **Target URL**.
    *   Textarea for **Test Intent/Description** (e.g., "Verify login with valid credentials").
    *   "Generate Steps" button (Triggers AI generation).
3.  **Phase 2: AI Generation (Mock/Real)**:
    *   Frontend sends intent + URL to Backend API (New Endpoint: `POST /api/cases/generate`).
    *   Backend returns a proposed `TestCase` object (Name, URL, Steps).
    *   Frontend populates the Editor Form with this data.
4.  **Phase 3: Manual Refinement (Editor Form)**:
    *   Standard structured editor (as previously defined).
    *   User can modify the AI-generated Name, URL, and Steps.
    *   "Save" button commits the final case to the database (`POST /api/cases`).
    *   "Run" button executes the saved case.

## Technical Notes
*   **New Endpoint Needed**: `POST /api/cases/generate` which takes `{ url: string, intent: string }` and returns `{ name: string, steps: [...] }`.
*   **UI Flow**: List -> New Intent Modal -> Loading -> Editor View (Populated).
